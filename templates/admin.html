Metodología · ecuaciones y orden de los modelos

El resultado final es el promedio (ensemble) de 4 modelos que se complementan, en este orden:

1 · Logit→ 2 · XGBoost→ 3 · Random Forest→ 4 · Red Neuronal→ Ensemble
1 · Regresión Logística — base probabilística (peso + signo)
P(aprobar | x) = σ(β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ)
σ(z) = 1 / (1 + e−z) · función sigmoide (0→1, nunca colapsa a solo 0/1)
Se ajusta por máxima verosimilitud: ℓ(β) = Σ [ yᵢ·log(pᵢ) + (1−yᵢ)·log(1−pᵢ) ]. Cada βᵢ es el peso y su signo el signo esperado de la variable.
2 · XGBoost — predicción total con gradiente y hessiano
ŷ = Σk fk(x) · árboles aditivos
Obj = Σᵢ L(yᵢ, ŷᵢ) + Σk Ω(fk) · peso óptimo de hoja: w* = −G / (H + λ)
Usa el gradiente g (1ª derivada) y el hessiano h (2ª derivada) de la función objetivo → aprende rápido y preciso, con regularización Ω contra el sobreajuste.
3 · Random Forest — estabiliza y evita sobreajuste
P = (1/B) Σb=1B Tb(x) · promedio de B árboles (bagging)
Promediar muchos árboles distintos reduce la varianza → el aprendizaje queda lo más cercano y robusto posible.
4 · Red Neuronal — GELU en ocultas, Sigmoide en salida
a(l) = GELU( W(l) a(l−1) + b(l) ) · capas 31 → 64 → 32 → 1
GELU(z) = 0.5·z·(1 + tanh[ √(2/π) ( z + 0.044715 z³ ) ]) · salida: σ(z)
Las capas ocultas usan GELU (suave y derivable): acelera el descenso por gradiente y —a diferencia de ReLU— no anula neuronas a 0. La sigmoide solo en la salida da una probabilidad continua (no colapsa a 0/1 duro). Si una variable es más significativa, su peso W es mayor. Entrena por gradiente con early-stopping; abajo verificamos que aprende y que ninguna neurona queda muerta.

¿Cómo poner mi base de datos real o guardarla en la nube?
  1. Respaldar y Cargar en la Nube (Google Drive, etc.): puedes descargar el archivo SQLite completo janus.db usando el botón Descargar base SQL (.db) de arriba y subirlo a tu cuenta de Google Drive u otra nube para conservarlo seguro.
  2. Restaurar en producción: si tu servidor en la nube se reinicia o entra en suspensión y se limpia la base de datos (comportamiento habitual en servidores efímeros como Render gratis), simplemente usa el botón Restaurar base SQL (.db) para subir tu copia de seguridad de janus.db y recuperarás todas las evaluaciones e historial inmediatamente.
  3. Inspeccionar: descarga la base de datos y ábrela con DB Browser for SQLite en tu PC.
  4. Reentrenar con tu dataset: sube tu dataset real en Subir dataset y reentrenar (CSV/Excel con las columnas del modelo).
